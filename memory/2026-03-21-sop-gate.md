Okey sounds good. What is the difference between /new and /reset?
assistant: They're essentially the same — both aliases for starting a fresh session.

- `/reset` — clears context, starts fresh, no model change
- `/new [model]` — same as `/reset`, but optionally accepts a model name (e.g. `/new gpt-4o`)

Both:
- Create a new session ID
- Clear conversation history
- Re-read SOUL.md, AGENTS.md, USER.md, etc. (that's why the SOP changes will take effect)
- Run a short "hello" greeting turn to confirm

**When sent alone** (no other text), they just reset and greet. **When followed by text** (e.g. `/new check disk space`), the text becomes the first message in the new session.

So to pick up the SOP changes on existing sessions — just `/new` or `/reset` in each chat. No gateway restart needed.

Can give me simple over of what are most used commands of the telegram with openclaw? I only know like 5 of them. Want use it more efficiently
assistant: Here's the practical cheat sheet for Telegram:

**Daily drivers:**
| Command | What it does |
|---|---|
| `/status` | Model usage, session info, provider quota |
| `/new` or `/reset` | Fresh session (re-reads all config files) |
| `/stop` | Abort current run immediately |
| `/model` or `/models` | Switch model (picker or name) |
| `/context detail` | See what's eating your context window |

**Model & thinking:**
| Command | What it does |
|---|---|
| `/think off\|low\|medium\|high` | Toggle reasoning depth |
| `/fast on\|off` | Skip tool results, faster replies |
| `/verbose on\|off` | More/less detail in responses |
| `/reasoning on\|off` | Show or hide reasoning chain |
| `/new gpt-4o` | Reset session with specific model |

**Sub-agents:**
| Command | What it does |
|---|---|
| `/subagents list` | See running sub-agents |
| `/kill all` | Abort all running sub-agents |
| `/kill <id>` | Kill specific sub-agent |
| `/steer <id> <msg>` | Redirect a running sub-agent |

**Usage & debugging:**
| Command | What it does |
|---|---|
| `/usage full` | Token count + cost per reply |
| `/usage cost` | Session cost summary |
| `/export` | Export session to HTML |
| `/whoami` or `/id` | Your sender ID |

**Quick tips:**
- `/status` works inline — type `hey /status what's next` and it runs the command, sends the rest to the model
- `/think high` → deeper reasoning for hard questions, `/think off` → fast for simple stuff
- `/new` alone = reset + greeting. `/new check disk` = reset + immediately ask
