# 2026-03-22 — Session Summary (Evening Session)

## Volume Quota / Rate Limiting Investigation & Fixes

### Root Cause
- Lighter DEX "didn't use volume quota" = exchange-side rate limit, not client-side
- New premium accounts start at 1,000 TX quota, earn +1 TX per $5 volume
- Bot burned through quota rapidly — SL orders re-submitted every ~76s per position
- AI trader kill switch counted failed quota attempts as orders → close-all loop

### Fixes Implemented

**1. SL order handling** (branch `fix/sl-rate-limit-handling`)
- SL rate-limited orders now return False instead of True (was silently pretending success)
- Added quota remaining to rate-limit logs

**2. Quota tracking & tier verification** (branch `feature/quota-tracking`)
- Tracks `volume_quota_remaining` from exchange responses
- Verifies account tier on startup via accountLimits API
- Premium confirmed: `user_tier = premium`, 0 LIT staked (4,000/min tx rate)

**3. Quota-aware cooldown** (branch `feature/quota-cooldown`)
- Exponential backoff: 60s → 120s → 300s max
- 16s pacing window when quota < 50
- `VolumeQuotaError` exception for clean handling

**4. Quota prioritization** (branch `feature/quota-prioritization`)
- Skip new opens when quota < 50
- Emergency mode (quota < 10): SL-only orders

**5. Graduated SL retry delays** (branch `fix/sl-retry-backoff`)
- 1st fail → 15s, 2nd → 1min, 3rd → 5min, 4th+ → 15min + alert
- Applied to DSL, legacy, and AI close paths

**6. Trailing floor** (branch `feature/trailing-floor`)
- Fixed ROE buffer from HW instead of percentage-based lock_hw_pct
- Buffers: 5% (tier 1), 4% (tier 2), 3% (tier 3), 2% (tier 4)
- URA example: floor at +20.2% instead of +18.9% (1.3% tighter)

**7. Alert fixes** (branch `fix/close-alert-consistency`)
- DSL closes now get post-close completion alerts
- Telegram 429s get logged instead of silently dropped
- Per-position close_all alerts
- "Exit" → "Trigger" label in DSL alerts
- 20-minute periodic quota status alerts

**8. AI trader kill switch fix**
- `max_orders_per_hour`: 12 → 50
- `max_rejection_halt_count`: 5 → 15
- Cleared 5 stale rejection entries from DB

**9. Bug fix: return → continue**
- In `_process_signals`, `return` in quota/cooldown checks skipped all remaining signals
- Fixed to `continue` so remaining opportunities are still processed
