# 2026-03-22 — Session Summary

## Major Bugs Found & Fixed

### 1. Side Detection Bug (Critical)
- **Problem:** Bot used `pos.position > 0` to determine long/short, but Lighter's position size is **always positive**
- **Impact:** ALL short positions tracked as long → DSL logic inverted → profitable shorts hard-stopped as losses
- **Fix:** Use `pos.sign` field (+1 = long, -1 = short) in `bot.py`
- **Branch:** Fixed in commit `6b94de8`

### 2. DSL Close Loop (No Circuit Breaker)
- **Problem:** When close orders failed (volume quota), bot retried every ~15 seconds forever
- **Fix:** Added max 3 attempts + 15-minute cooldown + Telegram alert for manual intervention

### 3. Order Execution Method
- **Problem:** `create_market_order_limited_slippage` fires blindly; accepted (code=200) but not executed ("didn't use volume quota")
- **Fix:** Switched to `create_market_order_if_slippage` which pre-checks order book
- **Additional:** Rate-limit detection — don't log "success" when response says "didn't use volume quota"
- **Note:** Later reverted back to `create_market_order_limited_slippage` (2% slippage) per John's decision — if_slippage pre-check was causing issues too

### 4. AI Trader API URL (Critical)
- **Problem:** Config had `api.kilo.ai/v1` → all LLM calls failed with HTTP 404
- **Correct:** `api.kilo.ai/api/gateway`
- **Impact:** AI trader never made decisions, bot in AI mode never opened positions
- **Fix:** Updated URL in config
