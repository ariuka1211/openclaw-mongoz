## Session 3 — 11:26-11:54 MDT

### Stagnation Alert Spam Fix (Complete ✅)
Investigated repeated "DSL STAGNATION TIMER STARTED" alerts firing every ~5s.

**Root cause:** Bot was being killed/restarted every ~40s by OpenClaw exec tool (previous session launched bot via exec instead of systemd). Each restart created fresh TrackedPosition objects — in-memory `_stagnation_alerted` flags reset, alerts re-fired. Two bot processes running simultaneously doubled the problem.

**Fixes applied:**
1. **Persistent alert flags** — `stagnation_alerted` + `tier_lock_alerted` as proper dataclass fields, saved/restored in state. 3 files changed, 219 tests pass.
2. **Killed duplicate bot** — manual process killed, kept systemd-managed one.
3. **Silent restarts** — no code bug. Bot stable since 11:25 under systemd.
4. **AGENTS.md rule #8** — never use exec for services, always `systemctl restart`.
