# Phase 4: Split core/grid_manager.py into 3 files

## Goal
Split the 1,984-line `core/grid_manager.py` into 3 focused files:
- `core/capital.py` — Compounding, deployment, position recovery
- `core/order_manager.py` — Order placement, retry, fill matching
- `core/grid_manager.py` — State management, deploy orchestration, roll, check_fills

All methods stay on the GridManager class. Cross-file methods call via `self.<method>()`.

## CRITICAL RULES
1. **NO logic changes** — every function body must be IDENTICAL
2. **Methods stay on GridManager class** — use inheritance pattern
3. **Cross-file method calls work via `self`** (since they're all on the same class through inheritance)
4. **All imports** needed by methods must be at the top of each file (or inline within methods)
5. **GridManager** in grid_manager.py should inherit: `class GridManager(CapitalMixins, OrderMixin):`
6. `generate_levels_from_volume_profile` and `_generate_levels_from_bands` are `@staticmethod` — they're used by roll_grid AND grid_manager. If they're on GridManager they're fine.
